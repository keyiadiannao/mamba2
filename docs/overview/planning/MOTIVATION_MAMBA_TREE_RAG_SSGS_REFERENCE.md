## 1. 核心命题（为何是「架构层协同」）

| 观点 | 说明 | 批判性备注 |
|------|------|------------|
| **顺序递归 vs 树路径** | 沿根—叶（或 DFS）行走，本质是 **离散路径上的顺序状态更新**；与 SSM「\(h_t=f(h_{t-1},x_t)\)」的 **流式** 叙事一致。 | 本仓 **path-batch / `Mamba2PathReader`** 是 **玩具 trunk + 同树 harness**；**全因果预训栈** 见 **ENG-***、**§Ⅷ 门闩**，**分列**脚注。 |
| **固定大小隐状态 vs KV 随长增长** | 回溯若实现为 **快照拷贝/恢复**，**理论上** 比保留/重算 **大规模 KV** 更轻；**带宽/随机访问 KV** 在「多分支试探」叙事下常被强调为 Transformer 痛点。 | **本仓已测**：**§7**、**M1**、**G3-a/b** 对 **峰值/KV 字节/墙钟** 有 **分项 JSON**；**跨臂墙钟不对等** 必须脚注（见 **`benchmark_ssgs_vs_kv_tree_nav_wikitext`** docstring）。 |
| **内存访问「顺序一致」** | 沿单路径流式读入时，**顺序前向** 与 **SSM 递归** 契合；多分支则涉及 **快照与回退**，不再是纯顺序、且 **有工程成本**。 | **CPU/GPU 快照迁移**、**批量并行** 与 **单路径 micro-bench** 不对等 —— 见 **`ENGINEERING_NORTH_STAR_PLAN.md` §4.4**（Q1–Q5）与 **`PLAN_NOW_TO_DONE.md` §Ⅵ-0**。 |

## 2. 树状 RAG vs 平面 RAG；回溯何时值钱

**领域共识（外部工作；非本仓复现结果）**：

- **树状索引**（如 **RAPTOR** 类）：多抽象层级、根/中间/叶联合检索，有利于 **长文档、主题级、多跳** 类需求；文献报告在 **QuALITY / QASPER** 等上相对平面 chunk 检索有提升（具体数字以原论文为准）。
- **平面 top‑k**：工程简单，适合 **短上下文、单跳、低延迟**；与树状 **适用域不同**，比较时需 **同一任务与预算**。

**回溯（自我修正 / 重检索）**：

- **价值偏大**：多跳、多文档、检索噪声大、需 **试探—评估—换枝** 的场景；与 **Self-RAG、Adaptive-RAG、显式回溯修正** 等思路同族（实现未必用 SSM）。
- **可轻量化**：简单单轮可 **限制 1–2 次** restore/重检索，控制延迟与产品复杂度。

**本仓对应**：

- **机制证据**：**SSGS**（`dfs_ssgs_mamba` + 快照/恢复）、**M1 三臂**（SSGS vs TF-KV clone vs truncate）、**TASK §Ⅸ**（**分类任务** 与 **导航代价** **分列**）。
- **尚未被单一表格证明**：「Mamba + 树状 RAG + 回溯」在 **某一固定下游 benchmark 上全面 SOTA」—— 见 **`PLAN_NOW_TO_DONE.md` §Ⅸ-6**（**P1 Sprint 2（a）** 统一监督）。

## 3. 两大技术挑战与可能切入（研究化表述）

### 3.1 状态对齐（离散树 ↔ 连续隐空间）

- **问题**：层级父子关系是 **离散结构**；SSM 隐状态是 **连续压缩**，存在 **有损** 与 **语义对齐** 问题。
- **可能方向**（文献级）：**SSD / 结构化条件**（将路径或深度编码为对 **B/C** 或门控的 **条件输入**）、**稀疏掩码视角** 等 —— 属 **架构/训练** 创新，**超出** 当前 **玩具 reader + ridge 探针** 范围。
- **本仓边界**：**A2-S3**、**TASK 1c** 用 **路径向量 + 线性头** 做 **可行性** 与 **难度梯度**；**不**等同于 **已解决对齐**。

### 3.2 检索回溯（错枝后的恢复）

- **叙事**：在 **决策点** 保存 **小 footprint 快照**；走错则 **restore** 再试另一子枝；对比 **KV 截断/重算** 的成本叙事。
- **本仓实现线**：**`TensorNavState` / `MambaNavState`**、**`dfs_ssgs_mamba`**、**§7 S4**、**M1**；JSON 中 **`snapshots_taken` / `rollbacks`**。
- **必须正视的风险**（外部讨论亦强调）：
  1. **有损快照**：长序列下 SSM **细节丢失**，**restore 不一定等价于「从未走错」** —— 与本项目 **§3.5 三风险** 一致，需 **任务级** 或 **L3** 测量。
  2. **何时回溯**：需 **策略**（规则 / 分类头 / RL），否则易沦为 **纯工程 demo**。
  3. **H2D / 频繁拷贝**：快照若 **CPU 侧池化**，延迟可能 **吃掉** 理论优势 —— **Ⅵ-3 / F4** 方向。

| 风险 | 含义 |
|------|------|
| **有损状态** | 回溯语义 **不保证无损** → 需 **任务落差** 或 **L3 一致性** 测量。 |
